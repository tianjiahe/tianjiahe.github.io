我的github使用手册

序言

记录我 使用Github的一些操作过程

github的很多命令不常用,我也没时间去研究,这里仅针对我实际使用中的命令做记录

准备工作, 本地文件夹做成github仓库

选定一个目录作为git 仓库
这里选中 test文件夹后鼠标右键,打开 Git Bash here ,打开一个git控制台. 执行如下命令

1.  git init               '将当前文件夹初始化git，将其变成可管理的仓库；
                           '此时，test目录下会有一个“.git"的文件夹（是一个隐藏的文件夹），文件查看属性将其显示
2.    git add README.md      '将项目添加到仓库,若全部添加 git add . 即可
3.     git status             '可查看一些当前的添加的状态.
4.     git commit -m "first commit"  '将项目提交到github（注意步骤很重要，若忘记这一步骤，导致后面一直报错,记住先提交）
5.   ssh-keygen -t rsa -C "XXXXXX@qq.com   '本地仓库和远程的github是通过Https或者是SSH来连接的，所以要设置一下远程的仓库。
                                          创建ssh key. 邮箱是你gihub注册的邮箱地址）；
                                          然后在C盘的用户目录下一个.ssh文件夹(C:\Users\tianjiahe\.ssh)，打开里面有两个文件

6. 在github网站上,找到设置页面 https://github.com/settings/keys, 的 [SSH and GPG key](https://github.com/settings/keys)项.
   创建一个ssh Key,点击New SSh key.Title可以随便写，
   Key里的值将C盘下.ssh里的id_rsa.pub中的内容全部复制到这即可，最后点击Add SSh key。

7. 再在github上创建一个仓库,作为远程仓库. (Repository= test,实际上传地址:https://github.com/tianjiahe/test.git )

8. git remote add origin *    ' 将以上网址copy 到 * 处,    
                              ' git remote add origin https://github.com/tianjiahe/test.git 
9. git push -u origin master  '最后一步把提交后的文件推送上去.

以后再上传文件,执行如下命令

git add  filename
git commit -m "xxx" 
git remote add origin https://github.com/tianjiahe/test.git
git push -u origin master
